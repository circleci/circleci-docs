= CLI command migration guide
:page-platform: Cloud, Server v4+
:page-description: A mapping of legacy CircleCI CLI commands to their equivalents in CircleCI CLI v1.
:experimental:

This page maps legacy CircleCI CLI commands to their equivalents in the new link:https://cli.circleci.com/[CircleCI CLI v1]. Use it as a reference when updating scripts or workflows that use the legacy CLI.

Some legacy commands have been removed in v1. Where a command has no v1 equivalent, this is noted in the table.

[#config]
== Config

[.table-scroll]
--
[cols="2,3,3", options="header"]
|===
| Task | Legacy command | CLI v1 command

| Validate a config file
| `circleci config validate <path-to-config.yml>`
| `circleci config validate`

| Pack a multi-file config into a single YAML
| `circleci config pack <path>`
| `circleci config pack <path>`

| Process and expand a config file
| `circleci config process <path-to-config.yml>`
| `circleci config process <path>`

| Generate a config from a repository scan
| No equivalent
| `circleci config generate [path]`
|===
--

[#local-execution]
== Local job execution

[NOTE]
====
Local job execution with `circleci local execute` and `circleci build` has been removed in CLI v1. These commands ran jobs locally in Docker and are no longer supported.
====

[.table-scroll]
--
[cols="2,3,3", options="header"]
|===
| Task | Legacy command | CLI v1 command

| Run a job locally in Docker
| `circleci local execute <job-name>`
| Not available in v1

| Process a v2.1 config then run a job locally
| `circleci config process .circleci/config.yml > process.yml` +
`circleci local execute -c process.yml <job-name>`
| Not available in v1
|===
--

[#policy]
== Config policy management

[NOTE]
====
The `--owner-id` flag used in legacy policy commands has been renamed to `--org` in CLI v1.
====

[.table-scroll]
--
[cols="2,3,3", options="header"]
|===
| Task | Legacy command | CLI v1 command

| Evaluate a config against policies
| `circleci policy decide <path> --input <path>`
| `circleci policy decide --input <path> --org <org>`

| Compare local and remote policy bundles
| `circleci policy diff <path>`
| `circleci policy diff <path> --org <org>`

| Run a raw OPA evaluation locally
| `circleci policy eval <path> --input <path>`
| Not available in v1

| Fetch a policy bundle
| `circleci policy fetch [name] --owner-id <org-id>`
| `circleci policy fetch [name] --org <org>`

| Get policy decision logs
| `circleci policy logs [name] --owner-id <org-id>`
| `circleci policy logs [name] --org <org>`

| Push a policy bundle
| `circleci policy push <path>`
| `circleci policy push <path> --org <org>`

| Read policy settings
| `circleci policy settings --owner-id <org-id>`
| `circleci policy settings get --org <org>`

| Update policy settings
| `circleci policy settings --owner-id <org-id> --enable <true\|false>`
| `circleci policy settings set --org <org> --enabled`

| Test policies
| `circleci policy test <path>`
| Not available in v1
|===
--
