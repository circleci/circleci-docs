= CLI command migration guide
:page-platform: Cloud, Server v4+
:page-description: A mapping of legacy CircleCI CLI commands to their equivalents in CircleCI CLI v1.
:experimental:

This page maps legacy CircleCI CLI commands to their equivalents in the new link:https://cli.circleci.com/[CircleCI CLI v1]. Use it as a reference when updating scripts or workflows that use the legacy CLI.

Some legacy commands have been removed in v1. Where a command has no v1 equivalent, this is noted in the table.

[#terminology-changes]
== Terminology changes

CLI v1 reflects updated CircleCI terminology that resolves longstanding ambiguity in the word "pipeline". Two terms have changed meaning in ways that affect command names directly.

[.table-scroll]
--
[cols="1,1,2", options="header"]
|===
| Legacy | New | Definition

| Pipeline definition
| Pipeline
| A definition that specifies where your CircleCI configuration is stored and where your code is checked out from. A trigger references a pipeline and uses it when initiating a run.

| Pipeline
| Run
| The instance produced when an event matches a trigger. A run records the outcome of the attempt and groups any workflows that executed, including errors that occurred before workflows could start.
|===
--

As a result, commands that previously used `pipeline` to refer to a triggered execution now use `run`. For example, `circleci pipeline run` in the legacy CLI becomes `circleci run trigger` in v1, and commands for listing, watching, and cancelling triggered executions all live under `circleci run` rather than `circleci pipeline`.

[#new-in-v1]
== New in CLI v1

The following capabilities are new in CLI v1 and have no equivalent in the legacy CLI.

*Config*

* *Generate a config from a repository scan*: `circleci config generate [path]`.

*Pipeline runs*

* *List recent runs*: `circleci run list`.
* *Watch a run until completion*: `circleci run watch [<run-id>]`.
* *Cancel a run*: `circleci run cancel <run-number-or-id>`.
* *Rerun a workflow*: `circleci workflow rerun <workflow-id>`.
* *Rerun a workflow from the first failed job*: `circleci workflow rerun <workflow-id> --from-failed`.

*Jobs and artifacts*

* *List or download job artifacts*: `circleci artifact <job-id>`.
* *Get job details*: `circleci job get <job-id>`.
* *Get job step output*: `circleci job output get <job-id>`.
* *Open a job in the browser*: `circleci job open <job-id>`.

*Contexts*

* *List environment variables in a context*: `circleci context secret list <id|name>`.

*Project environment variables*

* *Delete a project environment variable*: `circleci envvar delete <name>`.

[#local-execution]
== Local job execution

[NOTE]
====
Local job execution with `circleci local execute` and `circleci build` has been removed in CLI v1. These commands ran jobs locally in Docker and are no longer supported.
====

[.table-scroll]
--
[.datatable,cols="2,3,3,1", options="header"]
|===
| Task | Legacy command | CLI v1 command | Notes

| Run a job locally in Docker
| `circleci local execute <job-name>`
| Not available in v1
|

| Process a v2.1 config then run a job locally
| `circleci config process .circleci/config.yml > process.yml` +
`circleci local execute -c process.yml <job-name>`
| Not available in v1
|
|===
--
