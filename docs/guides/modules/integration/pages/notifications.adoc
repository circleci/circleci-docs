Configure email notifications to receive notifications for builds, deployments, and Chunk usage. CircleCI also offers Slack notifications at the project level.

For communications during security incidents, deprecation notices, and other important org-wide communications, organization admins can also set up security and technical contacts for each organization.

== User-level notifications

Configure your notification preferences, including the email address to use and the notifications categories you would like to receive.

=== Default email address for notifications

CircleCI uses your "default" email address to send mandatory communications, which include:

* System and security messages.
* Billing and account updates.
* Deprecation notices.

=== Build notification preferences

Under "Build notification preferences" you can select between your saved email addresses per organization, or select `Don't send email notifications` for a specific organization as required.

If you are a member of multiple organizations, you can specify a different email address, if available, for each organization using the dropdown menus.

TIP: If you log in to CircleCI using an email and password then you will only have a single email address in this menu. If you log in to CircleCI via GitHub or Bitbucket then you will see all email addresses associated with your GitHub or Bitbucket account.

=== Notification categories

You can configure your preferences for notification _categories_ at menu:User settings[Notifications]. The following notification categories are available:

==== My work

The *My work* category allows you to receive an email every time a build you triggered finishes.

You can further filter the notifications you receive based on the status of the build as follows:

* On default branch (any status).
* On default branch (failed, error, unauthorized).
* Failed
* Succeeded
* Cancelled
* Unauthorized
* Error

NOTE: The "On default branch (any status)" setting overrides individual status filters. For example, if you have "Cancelled" notifications disabled but "On default branch (any status)" enabled under either "My Work" or "Followed Projects," you will still receive notifications for canceled workflows. This includes workflows that are automatically canceled when approval jobs expire after 90 days.

==== Deployment notifications

You can set up deployment notifications to receive notifications when a deployment using the release agent is inconclusive. For more information, see the xref:deploy:release-agent-overview.adoc[Release Agent Overview] guide.

==== Followed projects

The *Followed projects* category allows you to receive emails for any build that has finished in a project that you have followed.

You can further filter the notifications you receive based on the status of the build as follows:

* On default branch (any status).
* On default branch (failed, error, unauthorized).
* Failed
* Succeeded
* Cancelled
* Unauthorized
* Error

NOTE: The "On default branch (any status)" setting overrides individual status filters. For example, if you have "Cancelled" notifications disabled but "On default branch (any status)" enabled under either "My Work" or "Followed Projects," you will still receive notifications for canceled workflows. This includes workflows that are automatically canceled when approval jobs expire after 90 days.

==== Chunk notifications

You can set up chunk notifications to receive notifications when one of your teammates attempts to use Chunk to fix a an error or workflow. As org administrator you can then choose to enable Chunk for your organization. For a full guide to setting up and using Chunk, see the xref:guides:toolkit:chunk-setup-and-overview.adoc[Chunk Overview and Setup] guide.

== Organization-level contacts

Set organization-level contact email addresses. These contacts are used so that CircleCI has a designated point of contact for the following:

* *Security contacts*. Used during security events if no other key contact exists. A maximum of 5 email addresses can be stored.
* *Technical contacts*. Used for deprecation notices and platform-level changes directed at the organization. A maximum of 5 email addresses can be stored.

To set up organization-level contacts you must be an organization administrator. Follow these steps:

== Sending notifications with orbs

You can use orbs to integrate notifications into your configuration. CircleCI offers a Slack orb and an IRC orb. Several third-party orbs are also available. Search the link:https://circleci.com/developer/orbs?query=notification&filterBy=all[orb registry] to see what other orbs are available. For more information on using the Slack orb, see xref:getting-started:slack-orb-tutorial.adoc[Using the Slack Orb].
