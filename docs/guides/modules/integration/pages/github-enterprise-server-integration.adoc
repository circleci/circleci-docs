= GitHub Enterprise Server integration
:page-description: Connect your self-hosted GitHub Enterprise Server instance to CircleCI Cloud.
:experimental:
:icons: font

NOTE: The GitHub Enterprise Server integration is currently in **Beta**. Some features may not yet be available. For the latest information, check this page or contact your account team.

[#overview]
== Overview

CircleCI Cloud supports GitHub Enterprise Server as an add-on VCS integration. If your organization uses a self-hosted GitHub instance, you can connect it to CircleCI Cloud. You get the same CI/CD experience available to GitHub Cloud users, without needing to run CircleCI Server.

The integration uses the same GitHub App model as CircleCI's existing integration with GitHub Cloud. This includes pipeline triggering, project setup, commit status reporting, and all supported trigger event types.

[#supported-functionality]
== Supported functionality

* Pipeline triggering from GitHub events including push, pull request, and all other link:/docs/guides/orchestrate/github-trigger-event-options/[supported trigger event options].
* Custom webhook triggers.
* API triggers.
* Scheduled pipelines.
* Commit status reporting to your GitHub Enterprise Server pull requests.
* Pipeline definitions and triggers manageable via the CircleCI web app and API.
* HTTPS code checkout.
* Custom configuration file paths and cross-repo pipeline/trigger setups.
* Chunk, CircleCI's AI CI/CD agent.
* Rollback functionality.

[#not-yet-available]
== Not yet available

The following features are not yet available for the GitHub Enterprise Server integration:

* GitHub Checks

[#prerequisites]
== Prerequisites

Before starting, make sure you have:

* A GitHub Enterprise Server instance accessible over HTTPS.
* A CircleCI organization already set up (you do not need to create a new one).
* Admin access to at least one GitHub organization within your GitHub Enterprise Server instance.

[#organization-type]
== Organization type

The GitHub Enterprise Server integration is available to both `circleci` and `github` type organizations. We recommend using a `circleci` type organization. For details on how organization types work, see the link:/docs/guides/permissions-authentication/users-organizations-and-integrations-guide/[Users, organizations, and integrations guide].

[#connectivity]
== Connectivity

Your GitHub Enterprise Server instance must be accessible over HTTPS for CircleCI Cloud to connect to it.

If your instance is behind a firewall or not accessible over the public internet, contact your account team to discuss private connectivity options.

[#set-up-the-integration]
== Step 1: Register and install the CircleCI GitHub App

CircleCI connects to your GitHub Enterprise Server instance through a GitHub App. Setting up this connection involves registering a CircleCI App on your instance and then installing it on one or more GitHub organizations.

[#set-up-pipelines-and-triggers]
== Step 3: Set up pipelines and triggers

GitHub App Server pipelines support the same configuration source and checkout source flexibility as GitHub App pipelines for GitHub Cloud, including custom config file paths and cross-repo setups.

You can create and manage pipeline definitions and triggers through the CircleCI web app at **Project Settings > Project Setup**.

For details on trigger event options, see the link:/docs/guides/orchestrate/github-trigger-event-options/[GitHub trigger event options] page.

[#pipeline-values]
== Pipeline values

GitHub App Server pipelines use `github_server` as the value for `pipeline.integration_source`. All other pipeline values available to GitHub App pipelines are also available to GitHub App Server pipelines. For a full reference, see link:/docs/variables/[Project values and variables].
